遇到的问题

1、模板类的声明和实现需要放在同一个文件中，具体原因见 将模板类的声明和实现分别放在不同的文件中 。

2、使用友元函数重载 << 运算符时，不能在类内定义，类外实现，具体原因见 模板类中重载＜＜和＞＞操作符 。

更正：好像可以类内定义，类外实现，参考 C++ 矩阵类：逗号初始化 中的版本4，需要在类的定义前加上友元函数的声明。

3、使用 initializer_list 初始化列表实现不定长参数的构造函数。

4、实现拷贝构造函数时出现 ‘const Matrix<int, 2, 2>’ as ‘this’ argument discards qualifiers [-fpermissive] 错误，是因为参数是 const 类型，但在函数内部调用了非 const 的 row() 和 col()，有可能会破坏参数状态。因此，给 row()、col() 函数添加 const 修饰符。

5、执行Matrix<int, 2, 3>::Constant(num)时，使用 memset() 给 m_data 赋值时，如果 num=0，可以得到正确结果；如果 num=1，则矩阵元素都为16843009，这是因为 memset 是按字节填充数据1，而 int 占四字节，最终的值为00000001 00000001 00000001 00000001，转换为十进制为16843009。参考 关于memset函数初始化数组的“坑” 。

6、在实现矩阵乘法时，编译出现call to non-‘constexpr’ function错误，是因为Matrix<Scalar, Rows, mat.cols()> tmp中mat.cols()为变量，而内置类型的模板参数必须是常量值。因此给函数返回值添加 const 修饰符：const int cols() const { return Cols; }，错误依然存在，查阅资料发现 const 修饰的函数返回值是只读变量，本质还是变量，改用 constexpr：constexpr int cols() const { return Cols; }，编译成功，因为 constexpr 修饰的函数或者变量被看作其值是编译时就确定的。参考 C++11 constexpr和const的区别详解 ， const与constexpr 。
